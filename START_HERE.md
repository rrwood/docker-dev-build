# 🚀 START HERE - Docker Development Container

Welcome! This project gives you a fully configured development container with Claude CLI, SSH access, and optional Claude Code + Gemini integration.

## 🎯 What You Get

✅ Alpine Linux container with Python, Git, Vim, Nano, etc.
✅ SSH access with password or key authentication
✅ Optional Claude CLI installation
✅ Optional ngrok installation
✅ LiteLLM setup for free Claude Code (using Gemini API)
✅ Automatic SSH key management
✅ Portainer-ready configuration

## ⚡ Quick Start - Choose Your Path

### 🪟 Windows Users (Portainer) - RECOMMENDED

**Use the PowerShell script (no dependencies needed):**

```batch
setup-powershell.bat
```

This will:
- Ask you configuration questions
- Generate Dockerfile and docker-compose.yml
- Optionally create SSH keys
- Create a Portainer deployment guide

Then deploy in Portainer UI and run `.\copy-ssh-keys.ps1`

**📖 See: [PORTAINER_QUICK_START.md](PORTAINER_QUICK_START.md)**

---

### 🐧 Linux/Mac Users (Auto-Deploy)

**Use the Bash script:**

```bash
./setup.sh
```

This will:
- Ask you configuration questions
- Generate Dockerfile and docker-compose.yml
- Build the container
- Start the container
- Copy SSH keys
- Test the connection

Everything is automatic!

**📖 See: [QUICKSTART.md](QUICKSTART.md)**

---

### 🪟 Windows Users with WSL/Git Bash (Auto-Deploy)

**Use the Bash script wrapper:**

```batch
setup.bat
```

Same as Linux/Mac - fully automatic!

**📖 See: [QUICKSTART.md](QUICKSTART.md)**

## 🎨 Customization Options

Both setup scripts will ask you:

1. **Container User**
   - Username for SSH access
   - Password (securely entered)

2. **Network Settings**
   - Container hostname
   - IP address
   - Docker network name

3. **Optional Components**
   - Install Claude CLI? (pre-install vs install later)
   - Install ngrok? (pre-install vs install later)

4. **SSH Configuration**
   - Generate SSH key pair?
   - Update ~/.ssh/config?

## 🔑 SSH Access

After setup, connect to your container:

```bash
# If you configured SSH keys
ssh <container-hostname>

# Or directly
ssh <username>@<container-ip>
```

Example:
```bash
ssh docker-dev
# or
ssh devuser@192.168.111.15
```

## 🤖 Claude Code Setup (Free with Gemini)

Once inside the container, you can set up Claude Code to use Google's Gemini API for free:

```bash
# 1. Setup LiteLLM
setup-litellm

# 2. Add your Google API key
nano ~/.config/litellm/.env
# Get your key from: https://aistudio.google.com/app/apikey

# 3. Start LiteLLM proxy (Terminal 1)
~/.config/litellm/start-litellm.sh

# 4. Export Claude environment (Terminal 2)
source ~/.config/litellm/export-claude-env.sh

# 5. Run Claude Code
claude
```

This lets you use Claude Code without paying for Anthropic's API!

## 🆘 Troubleshooting

### Container won't start
```bash
# Check logs
docker logs <container-name>

# Rebuild
docker-compose build --no-cache
docker-compose up -d
```

### Can't SSH to container
```bash
# Verify container is running
docker ps

# Test SSH service
docker exec <container-name> rc-service sshd status

# Re-copy SSH keys
.\copy-ssh-keys.ps1    # PowerShell
./copy-ssh-keys.sh     # Bash
```

### Need to reconfigure
Just run the setup script again! It will regenerate all files.
