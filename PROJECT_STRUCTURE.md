## Workflow

### Initial Setup

```
1. Run setup.sh
   ↓
2. Answer prompts (user, password, components, SSH)
   ↓
3. Generates: Dockerfile, docker-compose.yml, SSH keys
   ↓
4. Builds and starts container
   ↓
5. Copies SSH keys to container
   ↓
6. Tests SSH connection
   ↓
7. Done! SSH into container
```

### Inside Container Workflow

```
1. SSH into container
   ↓
2. (Optional) Run setup-litellm
   ↓
3. Add Google API key to ~/.config/litellm/.env
   ↓
4. Start LiteLLM proxy: ~/.config/litellm/start-litellm.sh
   ↓
5. Export Claude env: source ~/.config/litellm/export-claude-env.sh
   ↓
6. Run Claude Code: claude
   ↓
7. Start coding!
```

## Files Generated by Setup

When you run `setup.sh`, it creates/modifies:

### On Host Machine

- ✏️ `Dockerfile` - Customized based on your choices
- ✏️ `docker-compose.yml` - Updated with hostname/IP
- 🔑 `~/.ssh/docker-dev-container` - SSH private key
- 🔑 `~/.ssh/docker-dev-container.pub` - SSH public key
- ⚙️ `~/.ssh/config` - SSH connection config
- 📄 `workspace/.ssh/authorized_keys` - Copy of public key
- 🔧 `copy-ssh-keys.sh` - Script to copy keys to container

### Inside Container (after build)

The setup process ensures these are available:

- 👤 User account with sudo access
- 🔑 SSH keys in `~/.ssh/authorized_keys`
- 🛠️ Helper scripts in `/usr/local/bin/`:
  - `install-ngrok`
  - `setup-claude`
  - `setup-litellm`

### After Running setup-litellm (inside container)

- 📁 `~/.config/litellm/` directory
- ⚙️ `~/.config/litellm/litellm_config.yaml` - LiteLLM config
- 🔐 `~/.config/litellm/.env` - API keys
- 📝 `~/.config/litellm/.env.example` - Template
- 🚀 `~/.config/litellm/start-litellm.sh` - Start proxy
- 📤 `~/.config/litellm/export-claude-env.sh` - Export env vars

## Quick Reference

### Essential Commands

```bash
# Host machine
./setup.sh                      # Initial setup
./verify-setup.sh               # Verify setup
docker-compose up -d            # Start container
docker-compose down             # Stop container
docker-compose build            # Rebuild container
ssh <container-hostname>        # Connect to container

# Inside container
setup-claude                    # Install Claude CLI
install-ngrok                   # Install ngrok
setup-litellm                   # Setup Claude + Gemini
~/.config/litellm/start-litellm.sh  # Start LiteLLM proxy
source ~/.config/litellm/export-claude-env.sh  # Export Claude env
claude                          # Run Claude Code
```

## Customization

### Modify Container Configuration

1. Edit `Dockerfile` or `docker-compose.yml` manually
2. Or run `./setup.sh` again to regenerate

### Add More Helper Scripts

1. Create script in `scripts/` directory
2. Add to Dockerfile:
   ```dockerfile
   COPY scripts/my-script.sh /usr/local/bin/my-script
   RUN chmod +x /usr/local/bin/my-script
   ```
3. Rebuild: `docker-compose build`

### Change Network Settings

Edit `docker-compose.yml`:
```yaml
networks:
  dev-macvlan:
    driver: macvlan
    driver_opts:
      parent: eth0  # Your network interface
    ipam:
      config:
        - subnet: 192.168.111.0/24
          gateway: 192.168.111.1
```

## Version Control

### What to Commit

✅ Commit these:
- `scripts/` - Helper scripts
- `README.md`, `QUICKSTART.md`, etc. - Documentation
- `.env.example` - Template only
- `.gitignore` - Protect secrets

❌ Don't commit these (already in `.gitignore`):
- `.env` - Contains secrets
- `workspace/` - Your working files
- `*.key`, `*.pem` - SSH keys
- `copy-ssh-keys.sh` - Generated file

### Sharing the Setup

To share this setup:

1. Commit configuration scripts and docs
2. **Don't commit** generated Dockerfile/docker-compose.yml
3. Others run `./setup.sh` to generate their own config
4. Each user gets their own SSH keys and passwords

Or, if you want a standard setup:

1. Run `./setup.sh` with desired defaults
2. Commit the generated `Dockerfile` and `docker-compose.yml`
3. Others can use as-is or run `./setup.sh` to customize

## Troubleshooting Guide

| Problem | Solution | File to Check |
|---------|----------|---------------|
| Can't run setup.sh | Make executable: `chmod +x setup.sh` | setup.sh |
| Setup script fails | Check Docker is running: `docker info` | - |
| SSH won't connect | Run verification: `./verify-setup.sh` | verify-setup.sh |
| Wrong password | Edit Dockerfile, rebuild container | Dockerfile |
| Wrong IP address | Edit docker-compose.yml, restart | docker-compose.yml |
| Missing helper scripts | Check scripts/ directory exists | scripts/ |
| LiteLLM won't start | Check inside container: `pip3 list \| grep litellm` | - |
| Claude won't connect | Check env vars: `env \| grep ANTHROPIC` | ~/.config/litellm/.env |
